# Plan to Compile Banana.py to Executable

## Overview
The project is a Python script (Banana.py) that encodes/decodes images into audio files and interacts with IPFS via Pinata. It requires numpy, soundfile, and requests libraries. The goal is to create standalone executables for Windows, Mac, and Linux using PyInstaller, packaged as single files.

## Limitations
PyInstaller does not support cross-compilation between operating systems natively. For Windows, we can use Docker with Wine emulation to build .exe files on Linux. For Mac, cross-compilation is complex and not reliably supported; use CI/CD or build on macOS.

Since the current environment is Linux, we can build Linux and Windows executables here using Docker for Windows. For Mac, provide instructions for building on macOS or using CI.

## Steps

### 1. Install Dependencies
- Install Python dependencies listed in Requirements.txt: numpy, soundfile, requests.
- Command: `pip install -r Requirements.txt`

### 2. Install PyInstaller
- Install PyInstaller globally or in a virtual environment.
- Command: `pip install pyinstaller`

### 3. Build Linux Executable
- Run PyInstaller with --onefile to create a single standalone binary.
- Command: `pyinstaller --onefile Banana.py`
- Output: `dist/Banana` (Linux executable)

### 4. Build Windows Executable (using Docker on Linux)
- Use Docker with cdrx/pyinstaller-windows image to build .exe via Wine emulation.
- Ensure Docker is installed.
- Run: `docker run -v "$(pwd):/src/" cdrx/pyinstaller-windows`
- This assumes Requirements.txt is present and will install dependencies.
- Output: `dist/windows/Banana.exe`

### 5. Build Mac Executable
- Recommended: Use GitHub Actions or similar CI/CD to build on macOS runners.
  - Create a GitHub repository with the code.
  - Add a workflow file (.github/workflows/build.yml) that:
    - Uses macOS runner (e.g., macos-latest)
    - Installs Python, dependencies (pip install -r Requirements.txt)
    - Installs PyInstaller (pip install pyinstaller)
    - Runs: pyinstaller --onefile Banana.py
    - Uploads the executable as artifact.
- Advanced (not recommended): Cross-compile using custom toolchains like osxcross, but this requires significant setup and may not work reliably for complex apps.

### 6. Testing
- Test each executable on its respective platform to ensure it works without Python installed.
- Verify that the script's functionality (encoding/decoding images, IPFS upload/download) operates correctly.

### 7. Distribution
- Provide the executables to users for their platforms.
- Note: The script requires user input via console, so the executables will open a terminal/console window.

## Notes
- If building on multiple platforms via CI/CD (e.g., GitHub Actions), configure workflows to run PyInstaller on each OS.
- No additional assets (like images) are bundled automatically; if needed, use --add-data flags in PyInstaller commands.
- The script does not appear malicious and is suitable for compilation.
